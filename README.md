## SendCloud是什么？

SendCloud是国内专业的云端邮件发送平台,为开发者和企业提供便捷的API接口来调用服务。说白了,它就像是邮件界的"快递公司"——你不用自己搭建复杂的邮件发送系统,也不用担心被各大邮箱门户拦截,把邮件交给SendCloud,它会根据不同邮箱的收件规则,找到效率最高、速度最快的路径,确保你的邮件准确送达。

这个平台现在已经服务上万家国内外企业,日发送量达到千万级别。而且它还有个更厉害的国际版——Aurora SendCloud,专门针对海外邮件发送做了优化,月发送量超过35亿封,送达率高达99.61%。

## SendCloud解决什么问题？

### 1. 国内邮箱送达率高

这是SendCloud最大的优势。很多人用过AWS SES或者Mailgun,发现国外邮件服务对QQ邮箱、网易邮箱这些国内主流邮箱的支持并不理想,经常被屏蔽或进垃圾箱。SendCloud专门针对国内邮箱做了优化,与各大邮箱门户建立了良好关系,可以申请白名单、处理误判申诉,确保邮件顺利送达。

有用户实测,同样的邮件内容,用AWS SES发送QQ邮箱基本全军覆没,换成SendCloud后送达率直接拉满。

### 2. 发送速度快

触发邮件平均3秒可达——这个速度对于注册验证、密码重置这类场景非常重要。用户注册完账号,等了半天收不到验证邮件,体验多差啊。SendCloud由于采用国内线路,API响应速度也非常快。

### 3. 功能全面

- **WEB API和SMTP两种接入方式**：想怎么接就怎么接
- **邮件模板管理**：支持变量替换,实现个性化发送
- **实时数据追踪**：邮件打开率、点击率、退订率一目了然
- **A/B测试功能**：测试哪个标题、哪个内容效果更好
- **地址列表管理**：批量发送更方便
- **DKIM签名、SPF验证**：提升邮件可信度,避免被当成垃圾邮件

### 4. 安全可靠

SendCloud提供完整的安全保障措施,包括SMTPS、HTTPS加密传输,邮址加密、账号加密、邮件内容加密等多重防护,最大程度降低数据泄露风险。还有集群化部署,确保服务稳定运行。

## 快速上手指南

### 第一步：注册账号
访问👉 [SendCloud官网](https://sendcloud.com)注册账号,会自动生成测试域名、测试API_USER供你测试。

### 第二步：配置发信域名
为了提升送达率,建议使用自己的域名(比如mail.yourdomain.com),然后配置SPF、DKIM、DMARC记录。SendCloud后台会给你详细的配置说明。

### 第三步：创建邮件模板
在SendCloud后台创建邮件模板,支持HTML格式,可以使用变量实现个性化(比如%name%、%code%等)。

### 第四步：调用API发送
SendCloud提供了多种语言的SDK,包括Python、PHP、Java、Node.js等。发送邮件只需几行代码：

python
import requests

url = "https://api.sendcloud.net/apiv2/mail/send"
params = {
    "apiUser": "your_api_user",
    "apiKey": "your_api_key",
    "from": "service@yourdomain.com",
    "fromName": "你的公司",
    "to": "user@example.com",
    "subject": "欢迎注册",
    "html": "感谢注册，你的验证码是：123456"
}

response = requests.post(url, data=params)
print(response.json())


### 第五步：查看数据报表
在SendCloud后台可以实时查看邮件发送情况、打开率、点击率等数据,帮助你优化邮件内容和发送策略。

## 一些实用技巧

1. **域名分离**：把发信域名和主域名分开,用子域名(如mail.xxx.com),这样即使域名被封也有回旋余地。

2. **独立IP**：如果预算充足,建议购买独立IP发信服务,被封几率更小,而且可以申请加入白名单。

3. **混合使用**：国内邮箱用SendCloud,国外邮箱用Mailgun,各取所长。

4. **申请提额**：免费用户可以发邮件到service@sendcloud.im申请提额,说明用途和原因,最高可提升到每天200封。

5. **预热发送**：刚开始使用新域名时,不要一上来就大量发送,要逐步增加发送量,建立域名信誉。
